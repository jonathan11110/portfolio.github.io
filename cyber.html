Creation of a honeypot

Honey pots are a common defence tactic used against attackers that have decided to launch an attack against the network. This is a computer system with the sole purpose of being sacrificed to the attacker, so it has built in security vulnerabilities and made to look valuable so that actually useful computers can remain safe. In addition to this, the honey pot can be used as a means to further enhance security by allowing the network administrator to learn about what attackers are using to attack the network without risking anything of value.
This is done with varying types of honeypots such as

  • Email traps:these bait spam emails with harmful links so that the senders IP address can be added to a deny list
  • decoy databases:these bait SQL injection and help monitor software vulnerabilities
  • malware honeypot:mimics software apps to invite malware attacks. This allows for malware analysis to be used to help develop countermeasures

By setting up a honeypot i will be able to use it to test means of attack, as well as ensure I am familiar with how honeypots work/look as i'm interested in pen testing
